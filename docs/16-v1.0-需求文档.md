# Seedance 素材包视频生成 v1.0 需求文档

## 1. 目标

交付一套可在 OpenWebUI 内稳定运行的“素材包驱动视频生成”能力，覆盖从聊天上传素材到任务查询的完整闭环。

核心目标：
- 用户通过聊天上传 ZIP（以及单文件）进入素材包体系。
- 用户在 prompt 中通过 `@素材名` 引用素材。
- 系统先校验引用，再提交 Seedance 生成。
- 用户可查询任务列表/任务状态并获取最终视频地址。

## 2. 版本范围

### 2.1 In Scope（v1.0）

- 素材包管理：创建、列表、详情、素材地址查询。
- 聊天上传入包：支持 ZIP；支持单媒体文件成包；支持同消息多单文件合并为一个包。
- 引用解析：`resolve_material_references` 严格校验。
- 视频生成：`generate_video_with_material_package` 与 `generate_and_wait_with_material_package`。
- 任务管理：任务列表、任务详情、轮询等待。
- 存储策略：TOS-only（素材引用统一走 TOS 对象地址/临时签名 URL）。

### 2.2 Out of Scope（v1.0 不做）

- 素材预览、时间轴编辑、可视化编排。
- 跨用户共享素材包。
- WebSocket/SSE 主动通知。

## 3. 名词约定（强制）

- `package_display_name`：用户识别名（ZIP 原名或系统命名）。
- `asset_package_id`：系统调用 ID（如 `pkg_xxx`）。
- 规则：
  - 对用户展示优先 `package_display_name`。
  - 工具与接口调用必须使用 `asset_package_id`。

## 4. 用户主流程

1. 用户在聊天中上传素材（ZIP 或单文件）。
2. LLM 调用 `create_material_package_from_chat_upload` 创建素材包。
3. LLM 向用户回报包信息（`asset_package_id`、`package_display_name`、可引用素材名）。
4. 用户给出视频需求，LLM 先调用 `resolve_material_references`。
5. 若有缺失引用，立即返回修正提示并停止。
6. 校验通过后，LLM 调用 `generate_and_wait_with_material_package` 或分步生成+查询。
7. 返回任务结果（含 `task_id/status/video_url` 或结构化错误）。

## 5. 功能需求

### FR-1 素材包创建与复用

- 支持 ZIP 入包、单文件入包、多单文件合并入包。
- 多单文件合并命名规则：`合并上传-n个素材-yyMMdd-HHmm`。
- 可按最新优先查看素材包列表并复用历史包。

### FR-2 包内素材查询

- 支持查询包内素材基础信息与地址信息：
  - `reference_name`
  - `relative_path`
  - `tos_key`
  - `tos_status`
  - 可选 `temp_url`

### FR-3 引用校验

- 从 prompt 提取 `@文件名`。
- 未命中时返回：`missing_references` + `available_references`。
- 引用缺失时禁止继续生成。

### FR-4 视频生成与任务管理

- 提交生成必须支持 Seedance 模型参数。
- 返回 `response_id/task_id`。
- 支持任务列表、单任务查询、等待终态。
- 成功返回 `video_url`；失败返回结构化错误。

### FR-5 错误结构统一

所有失败结果统一包含：
- `status_code`
- `error_code`
- `error_message`
- `request_id`（若可提取）

## 6. 接口基线（v1.0）

- `POST /api/v1/material-packages/`
- `POST /api/v1/material-packages/from-upload`
- `GET /api/v1/material-packages/`
- `GET /api/v1/material-packages/{package_id}`
- `GET /api/v1/material-packages/{package_id}/assets`
- `POST /api/v1/material-packages/{package_id}/resolve`
- `POST /api/v1/material-packages/{package_id}/generate`
- `GET /api/v1/material-packages/tasks`
- `GET /api/v1/material-packages/tasks/{task_id}`

## 7. 工具能力基线（v1.0）

- `create_material_package_from_chat_upload`
- `list_material_packages`
- `get_material_package`
- `get_material_package_assets`
- `resolve_material_references`
- `generate_video_with_material_package`
- `list_generation_tasks`
- `get_generation_task_status`
- `wait_generation_task`
- `generate_and_wait_with_material_package`

## 8. 非功能要求

- 安全：ZIP 路径穿越防护、隐藏文件过滤、类型校验。
- 稳定性：失败结果结构化、任务轮询可控。
- 可观测：日志可关联 `chat_id/asset_package_id/task_id/request_id`。
- 运维：支持 conda 运行环境和 systemd 常驻。

## 9. 验收标准（DoD）

1. 聊天上传后可稳定入包并可复用。
2. 引用缺失可准确阻断并提示修正。
3. 校验通过可稳定提交生成并返回 `task_id`。
4. 任务列表/详情可用，成功任务可拿到 `video_url`。
5. 失败场景返回结构化错误，不出现空泛提示。
