# 运行命令手册（OpenWebUI + 素材包 + Seedance）

本文档给出日常开发、联调、排障所需的最小命令集合。

## 1. 环境与启动

### 1.1 进入目录并激活 venv

```bash
cd /Users/lucas/Documents/openWebui
source .venv/bin/activate
pip install tos
```

### 1.2 启动 OpenWebUI

```bash
open-webui serve --host 127.0.0.1 --port 8080
```

### 1.3 停止 / 重启

```bash
pkill -f "open-webui serve"
open-webui serve --host 127.0.0.1 --port 8080
```

## 2. Seedance 必备环境变量

未配置时，调用生成接口会出现 `ARK_API_KEY is not configured`。

```bash
export ARK_API_KEY='你的真实密钥'
export ARK_BASE_URL='https://ark.cn-beijing.volces.com/api/v3'
# 可选：启用 TOS 作为 Seedance 素材 URL 提供层（推荐生产）
export MATERIAL_PACK_TOS_ENABLED='true'
export TOS_ACCESS_KEY='你的TOS AK'
export TOS_SECRET_KEY='你的TOS SK'
export TOS_ENDPOINT='tos-cn-beijing.volces.com'
export TOS_REGION='cn-beijing'
export TOS_BUCKET='你的bucket'
export TOS_PREFIX='material-packages'
export TOS_PRESIGN_EXPIRES_SECONDS='3600'
```

可选：写入你的 shell profile（`~/.zshrc`）实现持久化。

### 2.1 推荐文件化配置（可长期复用）

填写文件：
- `/Users/lucas/Documents/openWebui/config/ark.env`

启动命令：

```bash
bash /Users/lucas/Documents/openWebui/scripts/run_openwebui.sh
```

说明：
- 该脚本会自动读取 `config/ark.env` 并启动 WebUI。
- 如果 `ARK_API_KEY` 为空，脚本会直接报错并阻止启动。
- 素材包后端也会尝试从 `config/ark.env` 兜底读取 `ARK_API_KEY/ARK_BASE_URL`（重启后生效）。

## 5. ZIP 上传与全链路调试

### 5.1 上传素材包 ZIP

```bash
ZIP='/绝对路径/your_assets.zip'

curl -sS -X POST 'http://127.0.0.1:8080/api/v1/material-packages/?wait_for_processing=true' \
  -H "Authorization: Bearer $TOKEN" \
  -F "zip_file=@${ZIP};type=application/zip" \
  -F 'chat_id=test-chat'
```

### 5.2 查询单个素材包

```bash
PKG='pkg_xxxxxxxxxxxxxxxx'

curl -sS "http://127.0.0.1:8080/api/v1/material-packages/${PKG}" \
  -H "Authorization: Bearer $TOKEN"
```

### 5.2.1 从聊天上传文件创建素材包（from-upload）

```bash
# 先通过 /api/v1/files 上传获得 file_id，这里假设为 $FILE_ID
FILE_ID='xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx'

curl -sS -X POST 'http://127.0.0.1:8080/api/v1/material-packages/from-upload' \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"chat_id\":\"test-chat\",\"upload_ids\":[\"${FILE_ID}\"]}"
```

说明：
- `upload_ids` 可传多个。
- 多个单媒体文件会合并为一个包，并按规则命名：`合并上传-n个素材-yyMMdd-HHmm`。
- 若传 ZIP，需单独请求（不能与其他文件混传）。

### 5.2.2 查询素材包内资产地址

```bash
# 仅返回 tos_key / tos_status
curl -sS "http://127.0.0.1:8080/api/v1/material-packages/${PKG}/assets" \
  -H "Authorization: Bearer $TOKEN"

# 额外返回临时 URL（排障用）
curl -sS "http://127.0.0.1:8080/api/v1/material-packages/${PKG}/assets?include_temp_urls=true" \
  -H "Authorization: Bearer $TOKEN"
```

### 5.3 校验 prompt 引用

```bash
curl -sS -X POST "http://127.0.0.1:8080/api/v1/material-packages/${PKG}/resolve" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"使用 @intro.mp4 和 @voice.wav 生成 8 秒短片"}'
```

### 5.4 触发生成

```bash
curl -sS -X POST "http://127.0.0.1:8080/api/v1/material-packages/${PKG}/generate" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"使用 @intro.mp4 和 @voice.wav 生成 8 秒短片","model":"doubao-seedance-2-0-260128","duration":8,"ratio":"16:9","watermark":false}'
```

说明：
- `doubao-seedance*` 模型会自动走 `contents/generations/tasks`。
- 返回里的 `response_id` 即 `task_id`（用于查任务状态）。
- 当前实现是 TOS-only：仅通过 TOS 预签名 URL 提供素材。

### 5.4.1 启用 TOS 后的最小验证

先确保 `MATERIAL_PACK_TOS_ENABLED=true` 且 `TOS_*` 已配置，然后重启服务并重新上传 ZIP。  
之后可直接执行：

```bash
curl -sS -X POST "http://127.0.0.1:8080/api/v1/material-packages/${PKG}/generate" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"使用 @01_FR1.mp4 @1_00005.JPEG @EN2.mp3 生成 8 秒广告片","model":"doubao-seedance-2-0-260128","duration":8,"ratio":"16:9","watermark":false}'
```

若成功，返回会包含 `response_id`（任务 ID）。

### 5.5 查询任务状态

```bash
TASK_ID='task_xxxxxxxxx'

curl -sS "http://127.0.0.1:8080/api/v1/material-packages/tasks/${TASK_ID}" \
  -H "Authorization: Bearer $TOKEN"
```

### 5.5.1 查询任务列表（支持筛选）

```bash
# 最近 20 条
curl -sS "http://127.0.0.1:8080/api/v1/material-packages/tasks?limit=20" \
  -H "Authorization: Bearer $TOKEN"

# 按素材包筛选
curl -sS "http://127.0.0.1:8080/api/v1/material-packages/tasks?package_id=${PKG}&limit=20" \
  -H "Authorization: Bearer $TOKEN"

# 按状态筛选（running/succeeded/failed/...）
curl -sS "http://127.0.0.1:8080/api/v1/material-packages/tasks?status=running&limit=20" \
  -H "Authorization: Bearer $TOKEN"
```

### 5.6 等待任务到终态（shell 轮询）

```bash
TASK_ID='cgt_xxxxxxxxx'

for i in $(seq 1 120); do
  STATUS="$(curl -sS \"http://127.0.0.1:8080/api/v1/material-packages/tasks/${TASK_ID}\" \
    -H \"Authorization: Bearer $TOKEN\" | jq -r '.status // .raw_response.status // empty')"
  echo "[$i] status=${STATUS}"

  if [[ \"$STATUS\" == \"succeeded\" || \"$STATUS\" == \"completed\" || \"$STATUS\" == \"failed\" || \"$STATUS\" == \"error\" || \"$STATUS\" == \"cancelled\" ]]; then
    break
  fi
  sleep 3
done
```

## 6. ZIP 隐藏文件处理程序

### 6.1 服务端自动过滤（已内置）

上传接口会自动过滤以下条目，不会进入可引用素材列表：
- `__MACOSX/`
- `._*`
- `.DS_Store`
- `Thumbs.db`
- `desktop.ini`
- 其他点文件/点目录

### 6.2 本地清洗脚本（可选）

```bash
python scripts/sanitize_material_zip.py /path/input.zip --only-media -o /path/output.clean.zip
```

用途：
- 预处理用户 ZIP（演示/测试环境）
- 快速验证“脏文件比例”
- 复现线上过滤逻辑

## 8. 常见报错速查

- `Not authenticated`
  - token 无效/过期，重新登录并刷新 token。
- `ARK_API_KEY is not configured`
  - 服务端缺少 `ARK_API_KEY`。
- `packages: []`
  - 当前账号下尚未上传成功素材包，或上传时全失败。
- `missing_references`
  - prompt 中 `@文件名` 与素材包引用名不一致。
- `Unable to resolve TOS URL for some references`
  - 请检查 TOS AK/SK、Bucket 权限，以及素材包本地缓存文件是否仍存在。
- `TOS SDK is required but not installed`
  - 当前 venv 缺少 SDK，执行：`source .venv/bin/activate && pip install tos`
- `TOS is required for material package upload`
  - 当前是 TOS-only 模式；必须启用 `MATERIAL_PACK_TOS_ENABLED=true` 并配置 `TOS_*`。
- `GenerationTaskPollingTimeout`
  - 任务在超时时间内未进入终态；可增大 `wait_generation_task` 的 `timeout_seconds` 后重试查询。
